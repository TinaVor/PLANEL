# Настройка Telegram-бота для PLANEL

После настройки пользователи получают:
- **Утром (по умолчанию 09:00)** — список задач на сегодня
- **Днём (14:00)** — напоминание запустить Помодоро + прогресс дня
- **Вечером (21:00)** — приглашение на рефлексию + итоги дня
- Команды бота: `/today`, `/stats`, `/unlink`

Часовой пояс уведомлений — Europe/Moscow.

## 1. Создать бота

1. Открой Telegram, найди **@BotFather** → `/newbot` → задай имя и username (например `MyPlanelBot`).
2. BotFather пришлёт **token** вида `1234567890:AAFb...` — сохрани его.
3. (Опционально) `/setdescription`, `/setabouttext`, `/setuserpic`.

## 2. Применить SQL-миграцию

В Supabase Dashboard → SQL Editor → запусти содержимое [docs/telegram-setup.sql](telegram-setup.sql). Создаст таблицу `user_telegram` с RLS.

## 3. Задать переменные окружения

На Render → Web Service → **Environment** добавь:

| Ключ | Значение |
|---|---|
| `TELEGRAM_BOT_TOKEN` | `1234567890:AAFb…` (из шага 1) |
| `TELEGRAM_BOT_USERNAME` | `MyPlanelBot` (без `@`) |
| `TELEGRAM_WEBHOOK_URL` | `https://planel-1.onrender.com/api/telegram/webhook` (твой реальный домен Render) |
| `TELEGRAM_WEBHOOK_SECRET` | любая случайная строка (опционально, но рекомендуется) |

После сохранения Render автоматически перезапустится. На старте сервер сам зарегистрирует webhook у Telegram (увидишь в логах `[tg/register] webhook set: …`) и запустит планировщик (`[tg/scheduler] start, every 60 sec`).

Локально для разработки: положи те же переменные в `.env`. Без `TELEGRAM_WEBHOOK_URL` бот не получит сообщений (Telegram не сможет до тебя достучаться), но scheduler будет работать.

## 4. Подключение пользователем

1. Пользователь открывает Профиль → карточка **Telegram-бот** → жмёт **«Получить код»**.
2. Получает 6-значный код (действует 15 мин) и кнопку «Открыть бота» (deep-link).
3. В боте отправляет `/start <код>` (или просто кликает по deep-link — Telegram сам подставит).
4. Бот отвечает «Готово, аккаунт привязан». В Профиле статус меняется на «Подключён: @username».
5. Пользователь настраивает время трёх уведомлений и переключатель «Получать уведомления», сохраняет.

Кнопка **«Тестовое сообщение»** проверяет связь — присылает в Telegram сообщение «PLANEL: тестовое сообщение получено».

## 5. Команды бота

| Команда | Что делает |
|---|---|
| `/start <код>` | Привязывает Telegram-аккаунт к пользователю PLANEL |
| `/today` | Список незавершённых задач на сегодня |
| `/stats` | Прогресс по сегодняшним задачам (выполнено / всего) |
| `/unlink` | Отвязывает аккаунт |
| Любой текст без `/` | Бот напомнит список команд |

## Безопасность

- `TELEGRAM_WEBHOOK_SECRET` — Telegram передаёт его в заголовке `X-Telegram-Bot-Api-Secret-Token`; webhook отвергает всё остальное → защита от поддельных запросов.
- Привязочный код — 6 цифр, живёт 15 минут, одноразовый (после привязки сразу очищается).
- Таблица `user_telegram` под RLS, доступна только service_role'у — токенов и chat_id нет в клиенте.

## Замечания про Render free tier

Free tier «засыпает» после ~15 минут без запросов. Это значит scheduler перестаёт тикать, и утренние уведомления не уйдут, пока кто-то не разбудит сервер. Решения:
- Платный тариф Render (server всегда онлайн), или
- Внешний пинг-сервис (UptimeRobot и т.п.), который раз в 5–10 минут дёргает `/healthz`.
