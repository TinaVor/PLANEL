# PLANEL — план пользовательского тестирования

Этот чек-лист проходи вручную после каждого релиза. Считаем сценарий пройденным, когда все пункты ✅.

Тип тестов:
- **Юнит-тесты** чистых функций: `src/tests.html` (открой в браузере, смотри стат «Всего / Прошло / Упало»)
- **E2E через UI** — ручной проход по сценариям ниже

---

## 1. Онбординг нового пользователя

- [ ] Открываю `/` — лендинг загружается, без ошибок в консоли
- [ ] Клик «Зарегистрироваться» → открывается форма `/app?mode=signup`
- [ ] Заполняю ФИО, дату рождения, Telegram, email, пароль → «Создать аккаунт»
- [ ] Вижу экран «Проверьте почту»
- [ ] После подтверждения email → вхожу логином + паролем
- [ ] Попадаю в раздел **Сегодня**, вижу дату, пустые блоки задач/привычек/еды/эмоций

## 2. Создание задачи

- [ ] В **Задачи и доски** нажать «Новая задача»
- [ ] Заполнить название, приоритет high, дедлайн «сегодня», статус «in_progress», сфера «Здоровье»
- [ ] Сохранить → тост «Задача создана», задача появляется в Списке
- [ ] Проверить бейдж «🫀 Здоровье» в карточке
- [ ] В **Сегодня** задача отображается в «Задачах на сегодня»
- [ ] В **Планер → Месяц** сегодняшняя дата помечена прогресс-баром и подписью «1 задача»

## 3. Kanban drag & drop

- [ ] Вкладка «Доска» → 3 колонки (Сделать / В процессе / Выполнено)
- [ ] Перетащить задачу из «Сделать» в «Выполнено» → счётчики обновились, задача зачёркнута на дашборде, в метриках `done` +1

## 4. Матрица Эйзенхауэра

- [ ] Создать 4 задачи: high+today, high+далёкое, medium+today, low без дедлайна
- [ ] Вкладка «Матрица» → по одной в каждом из 4 квадрантов

## 5. Проекты

- [ ] **Цели и проекты → Новый проект**: название, цвет, дедлайн, описание
- [ ] Открыть модалку задачи → в select «Проект» появился созданный проект
- [ ] Привязать к проекту 2 задачи
- [ ] На карточке проекта — прогресс «1/2 задач · 50% готово» (если одна done)
- [ ] Клик по карточке проекта → переход в «Задачи» с фильтром по проекту

## 6. Привычки

- [ ] Создать привычку «Утренний ритуал», сфера «Рост»
- [ ] Отметить сегодня + 2 предыдущих дня → бейдж «🔥 3 дн.»
- [ ] Переключатель **Неделя / Месяц**: 7 vs 30 квадратиков
- [ ] На **Сегодня** привычка видна в списке, чекбокс синхронизирован

## 7. Помодоро

- [ ] **Дашборд → Старт** — отсчёт пошёл, статус «Фокус-сессия идёт»
- [ ] Пауза → время замораживается
- [ ] Сброс → 25:00, «Готов к работе»
- [ ] Клик **?** → всплывает подсказка про технику
- [ ] Клик вне подсказки → закрылась

## 8. Календарь/Планер

- [ ] Переключить 4 режима: День / Неделя / Месяц / Год
- [ ] День: почасовая сетка 06:00–23:00
- [ ] Неделя: 7 колонок, сегодня подсвечен синим, задачи в нужных колонках
- [ ] Год: 12 мини-календарей; клик на месяц → Месяц; клик на день → День
- [ ] Кнопки ‹ › меняют диапазон контекстно; «Сегодня» возвращает

## 9. Колесо баланса

- [ ] Сдвинуть 7 слайдеров → радар перерисовывается; значения в localStorage
- [ ] Блок «Распределение» показывает количество задач/привычек по сферам

## 10. Дневник питания

- [ ] Задать цели 2000 ккал / 2500 мл
- [ ] Добавить запись: завтрак 450 ккал, 300 мл воды
- [ ] KPI: «450/2000», «300/2500», прогресс-бары на 22%/12%
- [ ] Удалить запись → KPI вернулись к 0
- [ ] График за 7 дней нарисован

## 11. 100 целей на год

- [ ] Добавить цель со всеми 6 полями
- [ ] Отметить «Достигнута» → прогресс «1 / 100», 1%
- [ ] Клик по **?** рядом с заголовком → попап SMART
- [ ] Редактировать цель, удалить

## 12. Дневник эмоций

- [ ] Заполнить: сон 7.5ч, энергия 8, настроение 😄, событие, мысль, благодарность
- [ ] Сохранить → в списке история появляется запись за сегодня
- [ ] Повторное сохранение — обновляет запись (не создаёт дубликат)
- [ ] На **Сегодня** в блоке «Эмоции» отображается сводка «😄 · 💤 7.5ч · ⚡ 8/10»

## 13. Заметки

- [ ] «Новая заметка»: заголовок, текст, теги через запятую
- [ ] Сохранить → карточка появилась в сетке
- [ ] Поиск по заголовку, тексту, тегу — фильтрует
- [ ] Клик по карточке → открывает редактирование
- [ ] Удалить через подтверждение

## 14. Уведомления

- [ ] Войти как admin@admin / 1111 → попадаешь в Админ-панель
- [ ] Форма рассылки: тема, текст, «в кабинет»+«на email», «Все пользователи» → Отправить
- [ ] Выйти, зайти как обычный пользователь → колокольчик в шапке с бейджем «1»
- [ ] Открыть дропдаун → виден заголовок «Плановые работы», «только что · также на email»
- [ ] Клик по уведомлению → отметка как прочитанное, бейдж исчезает

## 15. Профиль

- [ ] Клик по аватару в хедере → переход в **Профиль**
- [ ] Поменять имя, email, пароль → тосты об успехе, поля сохранены
- [ ] Сотрудник: пригласить по email, назначить роль viewer
- [ ] В списке сотрудников бейдж «⏳ ожидание» / «✓ активен»
- [ ] Изменить роль → обновилось; удалить → сотрудник исчез
- [ ] Google Calendar: клик «Подключить» → popup OAuth → `ok` → бейдж подключения; тумблер «Google-события» на календаре работает

## 16. Безопасность / роли

- [ ] Войти как invited viewer; в шапке — **switcher** workspaces
- [ ] Выбрать чужое пространство → баннер «Вы в пространстве X, роль: Просмотр»
- [ ] Кнопки «Новая задача» **скрыты** (hidden-for-viewer)
- [ ] Toggle чекбокса задачи → тост «У вас роль Просмотр»
- [ ] Попытка POST `/api/tasks?owner_id=<owner>` → 403 `forbidden_viewer_cannot_write`

## 17. Админ-панель

- [ ] Метрики: Всего пользователей / Подтверждены / За 7 дней / Всего задач
- [ ] Список пользователей с email, именем, датой регистрации, статусом
- [ ] После рассылки в логах Render: `[BROADCAST]` и `[BROADCAST:email]` по каждому адресату

## 18. Резилентность

- [ ] Разорвать сеть в DevTools → Offline → создание задачи → тост «Сетевая ошибка»
- [ ] Восстановить сеть → следующая попытка проходит
- [ ] Обновить страницу → профиль + токен сохранены, заметки/привычки/цели восстановлены из localStorage
- [ ] Выйти из аккаунта → localStorage `planel_token/user` очищены, локальные данные другого пользователя не видны

## 19. Кросс-браузер / мобилка

- [ ] Ширина ≤640px: bottom-nav (Сегодня / Задачи / Планер / Заметки / Профиль), sidebar скрыт
- [ ] Аватар в шапке виден и кликабелен
- [ ] Модалки помещаются, формы скроллируются

## 20. Накопленные данные (smoke)

Запусти автотесты: открой `/tests` (или `/tests.html`). Ожидаемый результат:

- Всего: **~30+** утверждений
- Упало: **0**

Если что-то упало — собери строку «✖ группа · тест · reason» и исправь соответствующий участок кода.
