# PLANEL

Веб-приложение для планирования: задачи, цели, привычки, аналитика, дневник, карточки для запоминания, Zettelkasten и командная работа в одном месте.

## О проекте

PLANEL — это full-stack планировщик: фронтенд-SPA без сборки на ванильном JS плюс Node.js/Express бэкенд с Supabase. Единый HTML-файл приложения содержит всю клиентскую логику и кастомную дизайн-систему. Подходит и как прототип для UX-итераций, и как рабочий сервис: есть авторизация, Google Calendar, Telegram-бот, команды и офлайн-хранение данных.

### Разделы приложения

- **Сегодня** — задачи на день, помодоро-таймер, XP-геймификация, привычки и питание на сегодня
- **Дашборд** — агрегированный обзор, серии, еженедельная активность
- **Задачи** — список, Kanban (3 статуса), матрица Эйзенхауэра, фильтры по приоритету и проекту, подзадачи с инлайн-превью и чекбоксами, повторы (ежедневно / будни / еженедельно / ежемесячно), свайп на мобильном, автор задачи в командных workspace
- **Календарь** — день/неделя/месяц, интеграция Google Calendar (бета), двусторонняя синхронизация, авто-план
- **Проекты** — проекты с цветовой маркировкой, декомпозиция на задачи, прогресс-бары, **6 готовых шаблонов** (Запуск MVP, Маркетплейс, Контент-канал, Мероприятие, Переезд, Новый навык)
- **Привычки** — недельный и месячный журнал, серии, недельные/месячные цели, связывание с целями из 100; статистика окна (7/30 дней) переключается с вкладкой журнала
- **Дневник** — 4 режима: **Питание** (калории / макро / вода, график за 7 дней), **Эмоции** (настроение, энергия, сон), **Рефлексия** дня (что получилось / помешало / завтра), **Благодарность** (3 вещи + кому спасибо)
- **Заметки** — 3 режима: **Заметки** (маркдаун + теги), **Карточки** (Anki-style с SM-2 lite — интервальное повторение по 4-балльной шкале), **Zettelkasten** (атомарные заметки со связями `[[…]]`, backlinks, битые ссылки открывают форму нового Zettel)
- **Колесо баланса** — 7 сфер жизни, радарная диаграмма, авто-построение по фактической активности задач и привычек, адаптированный SVG с подписями сфер
- **100 целей на год** — 2 вкладки: **Цели** (SMART с 6 шаблонами, импорт/экспорт CSV) и **Жизнь в неделях** (адаптация Tim Urban "Life in Weeks" — сетка 90×52, 5 периодов жизни цветом, 12 типичных событий, различающихся по полу, подсветка текущей недели)
- **Награды** — уровни, XP, ачивки
- **Команда** — приглашения сотрудников, роли (Владелец / Ставить задачи / Просмотр), рабочие пространства
- **Корзина** — soft-delete для всех типов сущностей (задачи, заметки, карточки, Zettel, питание, рефлексии, благодарности, цели) с восстановлением
- **Профиль** — email / пароль, переключатель тёмной темы, Google Calendar, Telegram-бот, управление командой

### Интеграции

- **Google Calendar (бета)** — OAuth, пулл событий в календарь, пуш задач как событий, двусторонняя синхронизация
- **Telegram-бот** — утренние/дневные/вечерние напоминания, тестовое сообщение, расписание
- **Админ-панель** — метрики пользователей, рассылка уведомлений (для владельца)

### Горячие клавиши

- `⌘K` / `Ctrl+K` / `/` — глобальный поиск (задачи, заметки, проекты, цели, привычки, разделы приложения)
- `N` — новая задача
- `?` — подсказка по горячим

Кнопка поиска также закреплена в шапке приложения рядом с уведомлениями.

## Технологии

**Фронтенд**
- HTML5 + ванильный JavaScript (SPA-навигация без роутера и фреймворка, обёрнуто в IIFE)
- [Tailwind CSS](https://tailwindcss.com/) через CDN + плагин forms + кастомная CSS-система с CSS-переменными для тем
- Google Fonts: `Inter` (UI), `Manrope` (заголовки)
- SVG-иконки inline, без внешних библиотек
- `localStorage` для клиентского состояния (настройки, колесо, дневники, привычки, цели, карточки, Zettel, игрофикация) — все бакеты привязаны к userId
- Service Worker (`sw.js`) + PWA-манифест (`manifest.webmanifest`)

**Бэкенд**
- Node.js (>=18), Express
- Supabase (auth + БД задач, пользователей, команд, токенов интеграций)
- Отдельные route-модули под `api/` (auth, admin, google, team, telegram, tasks)
- Dockerfile для деплоя

## Дизайн-система

- **Палитра:** сине-голубая (`#1F76F2` — основной бренд, `#06B6D4` — поддерживающий cyan)
- **Темы:** светлая и тёмная; единственный переключатель — в профиле (раздел «Оформление»). Автоматически подхватывается во всех 13 разделах приложения
- **Контраст:** все тексты соответствуют WCAG AA
- **Сетка:** 8/12-колоночная, отступы кратны 4 px
- **Адаптив:** проверено на 375 (мобильный) / 768 (планшет) / 1280+ (десктоп). Мобильная навигация — нижняя, десктоп — левый сайдбар. Компоненты с крупными визуализациями (Колесо баланса, Жизнь в неделях) — с горизонтальной прокруткой на узких экранах

## Запуск

### Локально

```bash
npm install
npm run dev     # hot-reload (node --watch server.js)
# или
npm start       # node server.js
```

Сервис поднимается на `http://localhost:3000` (лендинг), `http://localhost:3000/app` (приложение), `http://localhost:3000/tests` (тесты).

### Переменные окружения

Приложение читает из `process.env`:

- `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_KEY` — для auth и БД
- `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `GOOGLE_REDIRECT_URI` — для Google Calendar
- `TELEGRAM_BOT_TOKEN` — для Telegram-уведомлений

### Миграции Supabase

Перед первым запуском выполните SQL-скрипты из `docs/*.sql` в Supabase SQL Editor (в любом порядке, все идемпотентные):

- `team-setup.sql` — таблица `team_members` для ролей и приглашений
- `tasks-created-by.sql` — колонка `created_by` у задач для трекинга автора в команде
- `google-tokens-table.sql` — хранение OAuth-токенов Google Calendar
- `telegram-setup.sql` — токены и расписание Telegram-бота

### Docker

```bash
docker build -t planel .
docker run -p 3000:3000 --env-file .env planel
```

## Тесты

Клиентские self-тесты на ванильном JS (без фреймворка) проверяют ключевую логику:

- **Базис**: дата/время, HTML-escape, localStorage, серии привычек, аналитика задач, матрица Эйзенхауэра, фильтр заметок
- **Еда**: агрегация калорий/воды за день
- **100 целей**: прогресс, SMART-шаблоны, голы
- **Markdown** (inline/блочный) и защита от XSS
- **Recurrence**: daily / weekdays / weekly / monthly
- **Habit targets**: недельные/месячные цели, процент выполнения
- **Emotions**: корреляция Пирсона настроения и энергии
- **CSV**: escape и parse с кавычками/запятыми
- **Корзина**: soft-delete / восстановление
- **Помодоро**: статы за день/неделю/всего
- **Food macros**: 4-9-4 ккал
- **Subtasks**: прогресс, toggle
- **Gamification**: XP, уровни, 12 ачивок, правила начисления
- **Goal decomposition** и **auto-schedule**
- **Google sync mapping**: pull/push маршрутизация, strip [PLANEL task] маркеров
- **Жизнь в неделях**: периоды, гендерные различия, вычисление прожитых недель
- **Life milestones**: совершеннолетие / свадьба / первый ребёнок / средняя жизнь — разные значения для мужчин и женщин
- **Flashcards SM-2 lite**: интервалы, ease factor, очередь «на повторение»
- **Zettelkasten**: парсинг `[[ссылок]]`, backlinks, битые ссылки
- **Task author badge**: логика показа автора (своя/чужая/легаси)
- **Diary modes**: 4 режима (питание / эмоции / рефлексия / благодарность)
- **Project templates**: apply, множественное применение, due_offset от сегодня

Запуск: откройте `http://localhost:3000/tests` в браузере. Все тесты выполняются в текущем окне с изолированным `localStorage`-префиксом `test_planel_`. На момент последней проверки — **212 тестов, все проходят**.
